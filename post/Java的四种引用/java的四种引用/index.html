Java的四种引用

1. 强引用

1.1 概念

被直接指向的对象实例，即使JVM的堆内存不足爆出OOM错误也不会释放强引用的对象。

1.2 强引用对象的回收

只有当对象无法通过GC Root找到（没有被引用），强引用的对象才可被回收

在对象当中重写finialize方法(在Java 9 之后已经被启用），当对象被回收之后即可被调用，便可通过观察finialize方法是否被调用从而判断对象是否被回收：

@Data //lombok：快速生成get set方法
public class Tiger {

    private String name;

    private int age;
    @Override
    @Deprecated
    protected void finalize() throws Throwable {
        System.out.println("Triger被回收了");
    }

}

测试：只有tiger指向对象实例，当把tiger赋为空时，对象实例就没有任何指向了，此时便会回收对象实例并触发finalize方法

    public static void main(String[] args) {
        Tiger tiger = new Tiger();
        System.out.println("强引用的对象（GC前）："+tiger.hashCode());
        tiger = null;
        System.gc();
    }

运行结果：

强引用的对象（GC前）：3524
Triger被回收了

当把tiger = null 这一行代码去掉，则不会回收该强引用的对象，此时运行结果只有：

强引用的对象（GC前）：3524

1.3 内存不存时是否会回收

1.3.1 调整堆内存上限

首先通过JVM参数，设置堆内存上限为20MB，设置步骤如下：

点击运行符 ：

点击Modify Run Configuration

选择Modify options

选择Add VM options

在VM options 处设置 -Xmx20M

1.3.2 测试

首先先创建一个大小为10MB的Byte，当创建另一个10MB大小的数组时，堆内存不足（除了这两个数组之外，肯定还有其他对象占用堆内存，因此此时总占用内存一定大于20MB），此时进行了垃圾回收，但是用于强引用对象仍然有被指向，无法回收，因此即使抛出OOM 异常也不会回收强引用对象：

    public static void main(String[] args) {
        //初始化一个10M的byte数组
        int size = 1024*1024*10;
        byte[] bytes = new byte[size];
        //手动触发gc
        System.gc();
        //再次初始化一个10M的byte数组，此时堆内存已经不足了
        byte[] bytes2 = new byte[size];
    }

运行结果：报OutOfMemoryError ，即OOM 错误

Exception in thread "main" java.lang.OutOfMemoryError: Java heap space
	at reference.RefTest.main(RefTest.java:7)

2. 软引用

2.1 概念

当堆内存不足时才会对软引用的对象进行垃圾回收。与强引用不同，软引用对象的指向是通过SoftReference来实现的。

2.2 实现

  • 同样使用先前的步骤修改JVM参数，使得堆内存上限为20MB。首先创建一个被软引用 的10MB大小的对象。

  • 证明堆内存充足时软引用对象不会被回收 ：接着手动触发gc（堆内存充足时），手动触发gc前后，软引用对象均没有被回收。

  • 证明堆内存不足时软引用对象会被回收 ：接着再创建一个被 强引用 的10MB的对象（不会被回收），此时引发了堆内存的不足，自动触发了垃圾回收，最终发现软引用的对象被清除了。说明当堆内存不足时会清除软引用对象，之后再创建其他强引用的对象。

    public static void main(String[] args) {
        int size = 1024*1024*10;
        //创建一个10MB的对象
//        byte[] bytes = new byte[size];
        SoftReference<byte[]> softReference = new SoftReference<>(new byte[size]);
        System.out.println("软引用的对象（GC前）："+softReference.get());
        //手动触发GC
        System.gc();
        System.out.println("软引用的对象（GC后）："+softReference.get());
        //再创建一个10MB的对象，引发堆内存不足
        byte[] bytes2 = new byte[size];
        System.out.println("软引用的对象（堆内存不足后）："+softReference.get());
    }

运行结果：

软引用的对象（GC前）：[B@7cc355be
软引用的对象（GC后）：[B@7cc355be
软引用的对象（堆内存不足后）：null

注意：若要使用软引用，则不能先创建一个强引用的对象，再将其放到SoftReference当中 ，因为这样本质上该对象还是强引用的，当堆内存不足时还是无法清除该对象，如以下例子：

    public static void main(String[] args) {
        int size = 1024*1024*10;
        //创建一个10MB的强引用对象
        byte[] bytes = new byte[size];
        SoftReference<byte[]> softReference = new SoftReference<>(bytes);
        System.out.println("软引用的对象（GC前）："+softReference.get());
        //手动触发GC
        System.gc();
        System.out.println("软引用的对象（GC后）："+softReference.get());
        //再创建一个10MB的对象，引发堆内存不足
        byte[] bytes2 = new byte[size];
        System.out.println("软引用的对象（堆内存不足后）："+softReference.get());
    }

运行结果：

软引用的对象（GC前）：[B@7cc355be
软引用的对象（GC后）：[B@7cc355be
Exception in thread "main" java.lang.OutOfMemoryError: Java heap space
	at reference.SoftRefTest.main(SoftRefTest.java:16)

3. 弱引用

3.1 概念

无论堆内存是否充足，只要触发gc 就会回收弱引用的对象。与软引用类似，弱引是靠WeakReference 实现的。

3.2 实现 

    public static void main(String[] args) {
        WeakReference<Tiger> weakReference = new WeakReference<Tiger>(new Tiger());
        System.out.println("弱引用的对象(GC前）"+weakReference.get());
        //手动触发gc
        System.gc();
        System.out.println("弱引用的对象(GC后）"+weakReference.get());
    }

4. 虚引用

4.1 概念

被虚引用的对象不会被马上回收，会进入引用队列ReferenceQueue当中 ，主要用于监控对象被垃圾回收（GC）的时机，以便在回收后执行一些清理操作。

虚引用的主要应用场景是​​对象回收前的清理工作​​（有时候不只是清理对象的内存，也可能清理与对象有关的其他内存），尤其是在管理堆外内存时。例如，Java NIO中的 DirectByteBuffer类就使用了虚引用来确保在堆内对象被回收时，其关联的堆外内存也能被正确释放。

4.2 特点

4.2.1 必须与引用队列一起使用

  • 虚引用必须与一个 ReferenceQueue配合使用。当垃圾回收器准备回收一个对象时，如果发现它还有虚引用，就会在回收该对象的内存​​之前​​，把这个虚引用加入到与之关联的引用队列中（被虚引用的对象总是在被GC回收后、内存被清理前进入引用队列）。

  • 因此当在引用队列看到某个对象时，说明该对象的内存已经被清理了，即该对象已经彻底消亡了，永远无法再得到该对象了

4.2.2 不影响生命周期

  • 生命周期 指的是影响对象被回收的时机，前面三种引用影响的生命周期分别是：强引用使得对象无论如何都不会被清理；软引用使得对象只有在堆内存不足时才会被清理；弱引用使得对象在每次GC时都会被清理。

  • 与另外三种引用不同，虚引用不会影响对象的生命周期，说明：当一个对象除了被虚引用之外，如果还有被其他引用，那么该对象的生命周期就是其他引用的周期 ；当一个对象只有被虚引用时，那么可以视为该对象没有被引用，即随时可被回收 （如果一个对象仅被虚引用关联，那么它对于GC Roots来说是不可达的，会被标记为可回收对象）

  • 因此才要说虚引用是四种引用中最弱的一种，相当于没有被引用（如将强引用的指针设置为空），可随时被清理。

4.2.3 无法通过PhantomReference获得对象

  • PhantomReference.get() 的源码：
public T get() {
        return null;
    }

  • 即get()方法始终为空。这意味着你​​无法​​通过虚引用重新获得被引用对象的强引用，它的存在​​仅仅是为了关联一个引用队列​​，以便对象被回收时收到通知。

  • 这样设置的原因是：用户将无法通过虚引用重新获得被引用对象的强引用来改变生命周期

4.3 实现

  • 此时不应该重写Tiger中的finialize方法，因为在GC回收 Tiger对象前，会调用其 finalize()方法，而当对象重写了 finalize()方法时，会导致虚引用的​​延迟入队​​。因此Tiger的内容应该为：

    @Data //lombok：快速生成get set方法
public class Tiger {

    private String name;

    private int age;
//    @Override
//    @Deprecated
//    protected void finalize() throws Throwable {
//        System.out.println("Triger被回收了");
//    }

}

  • 首先创建一个tiger对象，刚开始其既被强引用、又被虚引用。并测试无法通过PhantomReference获取虚引用对象（会返回null）

    ReferenceQueue<Tiger> referenceQueue = new ReferenceQueue<Tiger>();
        tiger = new Tiger();
        PhantomReference<Tiger> phantomReference = new PhantomReference<>(tiger,referenceQueue);
        System.out.println("直接访问虚引用的对象："+phantomReference.get());

  • 接着断开tiger对象的强引用，这下tiger只剩被虚引用了（不可达了），接着调用System.gc() 时可将其回收。

  • （可选）：可以利用Thread.sleep() 适当延时，保证虚引用对象被GC回收后能够进入引用队列当中:

            tiger = null;
        System.gc();
        try {
            Thread.sleep(1000);
        }catch (InterruptedException e) {
            e.printStackTrace();
        }

  • 最后再查看虚引用对象是否进入引用队列当中：

            Reference<? extends Tiger> reference = referenceQueue.remove();
        if(reference != null) {
            System.out.println("对象已被gc回收，进入引用队列："+reference);
            reference.clear();
        }

完整代码：

public class PhantomRefTest {
    public static Tiger tiger;
    public static void main(String[] args) throws InterruptedException {
        ReferenceQueue<Tiger> referenceQueue = new ReferenceQueue<Tiger>();
        tiger = new Tiger();
        PhantomReference<Tiger> phantomReference = new PhantomReference<>(tiger,referenceQueue);
        System.out.println("直接访问虚引用的对象："+phantomReference.get());
        tiger = null;
        System.gc();
        try {
            Thread.sleep(1000);
        }catch (InterruptedException e) {
            e.printStackTrace();
        }
        Reference<? extends Tiger> reference = referenceQueue.remove();
        if(reference != null) {
            System.out.println("对象已被gc回收，进入引用队列："+reference);
            reference.clear();
        }
    }
}

最终运行结果：

直接访问虚引用的对象：null
对象已被gc回收，进入引用队列：java.lang.ref.PhantomReference@5b6f7412
