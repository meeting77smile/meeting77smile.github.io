服务器项目部署

1. 内网穿透

1.1 目的

为了让他人也能通过指定的url访问到自己启动的项目

1.2 实现步骤

1.2.1 下载与配置

  • 搜索 https://www.cpolar.com ，进入cpolar官网。

  • 注册账号。

  • 注册登录之后，按照首页的提示进行安装和配置。

  • 获取Authtoken ：点击首页右上角的 “用户设置”——>点击 “重置Authtoken” ——>复制Authtoken进行配置。

  • 配置Authtoken：按下 win+R ，在cmd命令行中复制如下命令：

    cpolar authtoken (这里改为你自己的Authtoken)

  • 出现如下类似的结果说明配置成功：

  • 启动cpolar：以管理员的身份运行cmd ，输入如下命令：

    net start cpolar

  • 出现结果，说明已经启动成功：

1.2.2 使用Cpolar

  • 通过地址 http://127.0.0.1:9200 （或者点击桌面的图标）访问cpolar的管理界面，输入注册时的邮箱与密码即可访问：

  • 点击 隧道管理 ——> 创建隧道 ，即可实现本地地址与穿透地址的映射

1.2.3 示例

  • 在vscode自己写一个cpolar.html文件：

    <!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Document</title>
</head>
<body>
    <h1>Hello Cpolar!</h1>
</body>
</html>

  • 利用 Live Server 插件，右键点击 Open With Live Server，即可运行

  • 运行之后，可以看到此时我的html文件的的本地端口号为5500（不同的设备可能不一样）

  • 记下该端口号，在刚刚的 创建隧道 页面中的 本地地址部分 输入为5500端口号，并选择 随机域名 和 http 协议的选项，点击 创建。

  • 在 隧道列表 界面即可发现刚刚创建的列表：

  • 点击 启动 之后，在 状态 ——> 在线隧道列表 当中便可看到刚刚创建的隧道以及公网地址：

  • 访问对应的公网地址：

  • 再点击想要访问的文件，即可看到刚刚我们自己写的界面了，但这次是通过公网地址访问的，而不是本地的5500地址：

  • 在手机上访问该地址，发现也能成功！而若是通过本地的地址访问，则无法访问。

1.2.4 不足

  • 若将运行的网页程序关闭或者将电脑关机，此时他人将无法通过先前的公网地址访问到网页。如果我们想实现在电脑或者程序关闭之后，他人还能访问到我们的项目，应该怎么做呢？

2.云服务器部署

为了解决上述问题，我们可以将项目部署在云服务器上，只要我们的云服务不到期，项目便可以一直在服务器上运行。

2.1 注册阿里云账号

  • 访问阿里云官网，注册账号

  • 访问 https://university.aliyun.com/buycenter ，或搜索 学生用券中心-阿里云云工开物 ，学生认证之后即可领取300元/年 的无门槛抵扣金，用于购买云服务器。

  • 领券之后，选择最左边第一个（推荐）e实例，点击立即购买

  • 其中 操作系统 和 云市场镜像 部分需要注意一下（配置之后也可以更改的，放心），可以参考我的配置：

  • 配置好之后，点击 “立即购买” 即可。

2.2 使用云服务器

  • 购买完云服务器之后，回到阿里云官网的主页，点击右上角的“控制台”

  • 再点击左上角的三条横线，选择 云服务器ECS

  • 在 云服务器ECS 的实例 或者 概览 页面即可看见刚才购买成功的云服务器，点击 实例 的名称：

  • 在这里便可看到云服务器的各种信息，如云服务器的公网IP，也就是我们之后要部署项目的公网IP。

  • 一开始，我们需要为云服务器配置 安全组 ，通俗理解就是开放云服务器的端口号，运行外部访问对应的端口，点击 手动添加 以下是常见需要开放的端口号，大家可根据需要添加，其中宝塔面板的端口是一定要开启的，方便接下来的操作。

我们可以通过 宝塔面板 在云服务器上部署项目，接下来我将带大家简单入门一下。

  • 首先，我们需要获取虚拟机的用户名和密码，所以需要回到实例区域，点击 更多 ：

  • 接着点击 重置实例密码：

  • 这里不仅可以修改密码，还能获取虚拟主机的 登录名 ：

  • 记下用户名与密码之后。接着，还是在 云服务器ECS 界面，点击右上角的 远程连接 ：

  • 输入刚刚的用户名与密码之后，即可远程连接云服务器，由于我们选择的是Linux系统，因此显示的是如下界面：

  • 之后，搜索 宝塔面板官网，点击下载安装，将官方的脚本复制到远程终端当中：

  • 输入脚本之后，即可出现如下信息：

  • 根据图中的提示，我们需要在刚刚的安全组中放行8888端口。

  • 放行端口之后，根据给出的 username，password ，我们变可以访问给出的面板地址，我们选择外网面板地址

  • 输入 外网面板地址 之后，我们会进入宝塔面板的管理界面（如果无法进入可能是安全组未开放的缘故），通过该界面我们便可以便捷地将项目部署到云服务器上。

2.4 基于宝塔面板部署项目：

  • 在面板首页，点击 网站 选项

  • 选择添加 HTML 项目，将绑定域名 设置为你的云服务器的公网IP地址：

  • 创建完成之后，点击 项目，此处显示我们项目的端口为80，因此我们还需要在安全组处添加80端口，表示允许其他用户访问我们的网站。

  • 创建成功之后，点击进入 根目录/www/wwwroot/域名 目录，点击 上传，即可通过将本地文件上传至宝塔面板，在这里我将先前写的cpolar.html文件上传：

  • 上传之后，输入http://域名/cpolar.html，即可实现其他用户访问当前我们的网页，无论我们设备是否关机，都不影响访问：

  • 以后我们便可以通过云服务器来部署自己的项目，还可以搭建自己的博客等等。

3.远程连接终端

每次需要使用云服务器终端的时候，如果都需要登录阿里云官网远程连接，会不会太麻烦了；并且如果除了使用云服务器部署项目之外，还想要使用它来学习linux，那么有什么便捷的连接工具呢。有的，这就是我们接下来要说的远程终端连接工具——MobaXterm

3.1 MobaXterm介绍

  • 一款免费、功能齐全的远程终端连接工具，通过它便可以快速连接远程的主机，并且还支持将本地的文件上传到远程主机，还能够查看、修改、删除远程主机的文件。

3.2 下载

  • 进入官网，点击 Download——>下载免费版本——>点击Install edition

  • 下载完成之后，在主页面，点击右上角的Session ：

  • 点击 SSH ：

  • 在 Remote host 处，输入云服务器的公网IP地址。无需输入 Specify username 的内容，默认为root用户，点击 Ok即可。

  • 之后输入云服务器的用户名与自己设置的密码 ，即可完成远程的连接：

  • 在右侧的侧边栏，就是云服务器对应的文件。通过侧边栏，我们可以做到将本地的文件上传到远程主机，还能够查看、修改、删除远程主机的文件。

3.3 Linux常用命令

  • 复制或者粘贴文本：不能用ctrl + C和ctrl + V，应该使用鼠标右键来选择。

  • 切换上一条、下一条命令 ：按下键盘的上、下箭头

  • ls ：查看当前目录下的文件

  • ls -ll ：长格式显示文件和目录的详细信息，包括权限、所有者、大小、最后修改时间等。

  • ls -a :显示当前目录下的所有命令

  • cd 指定文件夹名：打开当前路径下的对应路径

  • cd /home/user/doc：切换到指定路径（/ 表示绝对路径)

  • cd .. ：返回上一级目录

  • clear ：情况当前窗口的所有内容（不是删除，只是不显示，往上滑动会发现内容还存在）

  • mkdir 指定文件夹名称 ：在当前路径下新建文件夹。

  • rmdir 指定文件夹名称 ：在当前路径下删除文件夹。

  • 快捷键：ctrl + c：退出当前命令

  • top :显示当前进程占用CPU的情况。

  • ps H -eo pid,tid,%cpu | grep 线程的PID ：查看进程中的线程信息 ，常用于CPU飙高时的排查。

  • vim 文件名 ：新建并查看修改文件，如：vim test.txt。

    • 使用vim修改文件时，按下 i ，即可修改文件

    • 修改完成后，按下ESC，即可退出修改模式

    • 要退出当前文件并保存：先按下英文冒号，再输入wq，再按下回车键，即可保存并退出当前文件

  • rm 指定文件名：删除当前路径下的指定文件

  • rm -f：不询问地强制删除当前路径下的指定文件。（rm用得好可以构造核弹命令 ，将整个系统的所有文件都删除，无法恢复！慎用）

  • 有的，那就是通过docker 来配环境。对比上述过程，在docker 中，只需要进行两个步骤即可完成MySQL的配置：通过docker pull拉取镜像、通过docker run允许镜像。

4.1 docker的安装

  • 首先要安装一个yum工具

    sudo yum install -y yum-utils device-mapper-persistent-data lvm2

  • 安装成功后，执行命令，配置docker的yum源：

    sudo yum-config-manager --add-repo https://mirrors.aliyun.com/docker-ce/linux/centos/docker-ce.repo

    sudo sed -i 's+download.docker.com+mirrors.aliyun.com/docker-ce+' /etc/yum.repos.d/docker-ce.repo

  • 更新yum，建立缓存

    sudo yum makecache

  • 执行命令，安装docker：

    yum install -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin

4.2 启动和校验

  • 启动docker：

    systemctl start docker

  • 停止docker：

    systemctl stop docker

  • 重启：

    systemctl restart docker

  • 设置开机自启

    systemctl enable docke

  • 执行docker ps命令，如果不报错，说明安装启动成功：

    docker ps

  • 上述命令如果在运行过程中没有显示任何结果，说明docker配置环境成功（linux系统中，不报错就代表成功）：

4.3 配置镜像加速

  • 为了加快docker的下载速度，还需要先配置镜像加速。

  • 在阿里云官网首页，点击上方导航栏的 产品 ——> 点击左侧边栏的 容器 ——> 选择 容器镜像服务ACR ：

  • 选择 管理控制台 ：

  • 选择左侧边栏的 镜像工具 ——> 镜像加速器 ，根据操作文档进行配置镜像加速器：

  • 依次运行以下命令：

    sudo mkdir -p /etc/docke

    sudo tee /etc/docker/daemon.json <<-'EOF'
{
  "registry-mirrors": ["这里换成你的加速器地址"]
}
EOF

    sudo systemctl daemon-reload

    sudo systemctl restart docker

  • 若此过程无输出结果，无报错，说明配置镜像成功。

4.4 docker常见命令

  • docker pull ：拉取指定镜像，如docker pull mysql:8.0 表示拉取8.0版本的mysql，若不指定版本，则默认为最新版本。

  • docker images ：查看当前已下载的镜像。

  • docker run ：安装并且运行已下载的镜像，从而创建出新的容器（镜像安装完之后就形成了容器）。

  • dockers ps ：查看当前正在运行的容器。

  • docker ps -a ：查看所有的容器，包括停止运行的。

  • docker restart ：重启容器。由于docker run每次运行时都会创建新的容器，因此启动已有容器应该使用docker restart而不是run，这个别搞混了。

4.5 利用docker配置MySQL

  • 拉取mysql镜像：

    sudo docker pull mysql:8.0

  • 验证镜像拉取成功：

    docker images

  • 利用镜像创建容器：

    sudo docker run -d \
  --name mysql \
  -p 3307:3306 \
  -e MYSQL_ROOT_PASSWORD=your_password \  # 替换为你的密码
  -v mysql_data:/var/lib/mysql \          # 数据持久化卷
  -v /host/config:/etc/mysql/conf.d \     # 挂载自定义配置目录（可选）
  --restart=always \
  mysql:8.0

  • 说明：

    • name ：表示自定义的容器名称

    • -p 3307:3306 ：表示将云服务器的3307端口映射为对外的3306端口，这样之后如果其他用户想远程使用云服务器上的mysql，就可以通过云服务器公网IP地址:3306来访问。使用3307是因为云服务器默认已经有mysql了，要避免冲突，使用如下命令即可查看云服务器3306端口的占用情况：

      sudo netstat -tulnp | grep 3306

      • 发现已经有默认的mysql占用mysql了：

        tcp6       0      0 :::3306                 :::*                    LISTEN      85101/mysqld

    • 记得将命令中的密码改为自己想要自定义的。说明：最好将上述命令复制到记事本当中进行修改，删除不必要的回车空格之后，再复制到终端中运行，否则直接复制到终端可能不小心将命令拆分开，导致执行失败：

  • 运行成功之后，使用如下ps命令，发现容器已经创建并且成功运行：

    docker ps

  • 通过docker启动MySQL之后，使用如下命令即可进入mysql：

    sudo docker exec -it mysql mysql -uroot -p

  • 按下回车后，输入刚刚设置的密码。这里要注意，密码是不可见的，直接输入自己的密码，再按下回车即可。

4.7 mysql基本使用

  • 首先要知道，sql语句是不区分大小写的，即输入大写和小写字母是等价的，甚至可以大小写混用。

    CREATE DATABASE mydb;

  • 显示所有数据库

    SHOW DATABASES;

  • 进入使用数据库

    use mydb(这里换为你的数据库)

  • 进入数据库之后，在对应数据库创建表。我们这里创建一个简单的学生表：

    CREATE TABLE students (
    id INT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(50) NOT NULL,
    age INT,
    class VARCHAR(20)
);

  • 插入单条数据

    INSERT INTO students (name, age, class) VALUES ('zhangsan', 18, '(1)');

  • 插入多条数据

    INSERT INTO students (name, age, class) VALUES 
('lisi', 17, '(2)'),
('wangwu', 19, '(1)');

  • 查询所有数据

    SELECT * FROM students;

  • 条件查询年龄大于18的学生

    SELECT * FROM students WHERE age > 18;

  • 修改单个字段（将张三的班级改为高一(3)班）

    UPDATE students SET class = '(3)' WHERE name = 'zhangsan';

  • 删除特定记录（删除王五的信息）

    DELETE FROM students WHERE name = 'wangwu';

  • 查看表结构

    DESC students;

  • 添加新字段

    ALTER TABLE students ADD COLUMN gender CHAR(1);

  • 删除表

    DROP TABLE IF EXISTS students;

  • 删除数据库

    drop database mydb;

  • 退出mysql

    exit;
